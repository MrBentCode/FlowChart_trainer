Блок-схема алгоритма – это графическое изображение алгоритма в виде связанных между собой с помощью стрелок и блоков – графических символов, каждый из которых соответствует одному шагу алгоритма. Внутри блока дается описание соответствующего действия. На рис. 1.1 приведена блок-схема алгоритма, состоящего из 2 шагов.

Рисунок 1.1– Блок-схема алгоритма из 2 шагов

Существует несколько основных блоков: блок начала/конца, блок действия, блок решения, блок ввода/вывода, блок-счетчик. Блок начала/конца служит для обозначения начала и конца алгоритма. Данный блок представлен на рис. 1.2.

Рисунок 1.2 – Блок начала/конца

Блок действия – основной блок алгоритма. Служит для описания одной или нескольких операций и выполнения обработки данных любого вида. Внутри фигуры записывается описание выполняемой операции. Данный блок представлен на рис. 1.3.

Рисунок 1.3 – Блок действия

Блок решения отображает решение или функцию переключательного типа с одним входом и двумя или более альтернативными выходами, из которых только один может быть выбран после вычисления условий, определенных внутри этого элемента. Вход в элемент обозначается линией, входящей обычно в верхнюю вершину элемента. Если выходов два или три, то обычно каждый выход обозначается линией, выходящей из оставшихся вершин (боковых и нижней). Если выходов больше трех, то их следует показывать одной линией, выходящей из вершины (чаще нижней) элемента, которая затем разветвляется. Соответствующие результаты вычислений могут записываться рядом с линиями, отображающими эти пути. Данный блок представлен на рис. 1.4.

Рисунок 1.4 – Блок решения

Блок ввода/вывода предназначен для ввода или вывода данных. Данный блок представлен на рис. 1.5.

Рисунок 1.5 – Блок ввода/вывода

Блок-счетчик предназначен для обозначения начала цикла и задания его параметров. Задается изменяемое число (счетчик), минимальное значение, максимальное значение и шаг, с которым счетчик будет изменяться. Данный блок представлен на рис. 1.6.

Рисунок 1.6 – Блок-счетчик
